[커밋 정보 확인하기]
git log : 커밋 정보를 보여준다.
git log --graph : 커밋정보를 그래프 형식으로 보여준다.
git log --oneline : 커밋정보를 커밋 ID와 메세지를 한 줄로 간단하게 보여준다.
git log --oneline --graph :  그래프 형식으로 한 줄로 간단하게 보여준다.
git log -숫자 : 가장 최근 커밋을 숫자만큼 보여준다.

[충돌]
같은 파일 내에서 같은 줄을 수정 · 삭제 · 변경한 후, 해당 대상을 하나로 합칠 떄 충돌이 발생한다.
충돌이 발생하면 직접 내용을 수정한 후 스테이지에 올려주고, 커밋을 해줘야한다.

-

[병합]
rebase
- base를 다시 설정한다. (base : 브랜치가 뻗어나가기 시작하는 커밋)
- 뿌리를 바꿔주고 싶은 브랜치에서 작업 후, git rebase 뿌리가 존재하는브랜치명 적용
- rebase 완료된 후에는 뿌리가 존재하는브랜치명에서 merge하여 합쳐주는 것도 가능

rebase충돌
충돌 내용을 수정한 후 add 하여 스테이지에 올린 후, git rebase --continue를 사용하여 리베이스를 진행한다.
